# Typography System

blog9yu.dev의 타이포그래피 시스템은 **시맨틱 토큰 기반**으로 일관된 텍스트 스타일을 제공합니다.

## Architecture

```
Font Tokens (원시 값)
  ↓
Typography Styles (조합된 스타일)
  ↓
Semantic Tokens (사용 목적)
  ↓
Utility Classes (Tailwind 스타일)
```

- **Font Tokens**: Font Size, Weight, Line Height, Letter Spacing
- **Typography Styles**: Display, Heading, Body, Label 조합
- **Semantic Tokens**: Page Title, Section Title, H1-H6, Body Text 등
- **Utility Classes**: `.text-display-lg`, `.text-heading-xl`, `.text-body-md` 등

## 사용 방법

### Utility Classes (권장)

```tsx
// Display
<h1 className="text-display-lg">큰 제목</h1>

// Heading
<h2 className="text-heading-xl">섹션 제목</h2>
<h3 className="text-heading-lg">소제목</h3>

// Body
<p className="text-body-md">본문 텍스트</p>
<span className="text-body-sm">보조 텍스트</span>

// Label
<label className="text-label-lg">라벨</label>
<span className="text-label-sm">캡션</span>
```

### CSS Variables

```css
.custom {
	font-size: var(--font-size-body-md);
	font-weight: var(--font-weight-medium);
	line-height: var(--line-height-normal);
	letter-spacing: var(--letter-spacing-normal);
}
```

### TypeScript

```typescript
import { typographyStyles, semanticTypography } from "@/shared/styles/foundations/typography";

// 스타일 객체로 사용
const titleStyle = typographyStyles.display.lg;

// 시맨틱 토큰 사용
const pageTitleStyle = semanticTypography.pageTitle;
```

## 규칙

1. **무조건 시맨틱 토큰 사용** - 하드코딩된 font-size, font-weight 금지
2. **Utility Class 우선** - `.text-heading-lg` 같은 정의된 클래스 사용
3. **일관성 유지** - Display, Heading, Body, Label 카테고리 준수
4. **반응형 고려** - 필요시 `sm:text-heading-xl` 같은 반응형 클래스 조합
